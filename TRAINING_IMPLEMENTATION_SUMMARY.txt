For long training runs, use nbconvert to execute headlessly:

  PowerShell> cd D:\ecg-research
  PowerShell> .\.venv1\Scripts\Activate.ps1
  PowerShell> jupyter nbconvert --to notebook --execute `
              notebooks/master_pipeline.ipynb `
              --ExecutePreprocessor.timeout=-1 `
              --output logs/run_$(Get-Date -Format 'yyyyMMdd_HHmmss').ipynb

This allows training to run in background without keeping notebook browser open.

Issue 1: GPU Out of Memory
  Symptoms: RuntimeError: CUDA out of memory
  Solution: In config cell, change:
    BATCH_SIZE_GPU = 16        # Was 64
    GRAD_ACCUM_STEPS = 4       # Was 1
    # or
    TARGET_SAMPLES = 2500      # Was 5000

Issue 2: Training Too Slow on CPU
  Symptoms: Hours remaining estimate very high
  Solution:
    - Run medium test first (5k samples)
    - Reduce EPOCHS to 5-10
    - Use cloud GPU (see runbook Section 14)
    - Set ECG_PREPROCESS_LIMIT = 5000

Issue 3: Low Validation F1 Score
  Symptoms: F1 < 0.5 after multiple epochs
  Solution:
    - Run pre-flight check (verify class balance)
    - Check label mapping coverage
    - Increase EPOCHS to 30-50
    - Try lower LR (5e-4 or 1e-4)
    - Experiment with SCHEDULER_TYPE

Issue 4: Preprocessing Files Not Found
  Symptoms: manifest.jsonl or splits.json missing
  Solution:
    - Run preprocessing cells (Section 5)
    - Verify DATASET_DIR path is correct
    - Check logs/unified_label_mapping.csv exists
    - Review pre-flight checklist output

Issue 5: Checkpoint Load Error
  Symptoms: RuntimeError when loading checkpoint
  Solution:
    - Delete corrupted checkpoint
    - Check PyTorch version compatibility
    - Restart kernel and re-run
